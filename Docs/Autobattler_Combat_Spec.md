# Autobattler Combat Spec

## 1. 목적
- 이 문서는 Autobattler 전투 플로우의 구현 기준 문서다.
- 현재는 1:1 테스트를 기준으로 하지만, 추후 최대 8인 전투 페이즈까지 확장할 수 있도록 작성한다.
- 수치 밸런스와 일부 규칙은 추후 변경될 수 있으므로, 구현은 데이터 중심 구조를 전제로 한다.

## 2. 구현 원칙
- 모든 전투 스탯 값은 추후 쉽게 수정할 수 있어야 한다.
- 전투 수치는 코드 상수 하드코딩보다 데이터 에셋 또는 설정 데이터에서 조정 가능해야 한다.
- 전투 로직과 UI 로직은 분리한다.
- 데미지 계산은 공통 함수 또는 공통 유틸리티로 분리한다.
- 피해 타입은 enum으로 관리한다.
- 현재 전투는 원본 유닛이 아니라 전투용 클론으로 진행한다.
- 전투 중 변동된 HP와 Mana는 라운드 종료 후 원본 유닛에 반영하지 않는다.
- Fusion Networked 프로퍼티는 `Spawned()` 이후에만 접근한다.
- 전투 후보 탐색 시 despawn 중이거나 네트워크 상태가 준비되지 않은 유닛은 제외한다.

## 3. 전투 단위
- 전투는 전투용 클론으로 진행한다.
- 원본 유닛은 전투 중 숨김 상태로 유지된다.
- 전투 종료 후 클론은 제거되고, 원본 유닛은 전투 전 상태로 복원된다.
- 전투 중 발생한 HP, Mana 변화는 원본에 반영되지 않는다.

## 4. 스탯 정의
각 유닛은 최소 다음 전투 스탯을 가진다.

- AttackRange: int
- MoveSpeed: float
- MaxHp: float
- AttackPower: float
- Armor: float
- MagicResist: float
- MaxMana: int
- AttackSpeed: float
- ManaGainOnAttack: float
- ManaGainOnHit: float

기본값 또는 초기 기준값은 아래와 같다.

- MoveSpeed 기본값: `1`
- AttackSpeed: 초당 공격 횟수
- ManaGainOnAttack 기본값: `20`
- ManaGainOnHit 기본값: `10`

현재 구현 반영 사항:
- `AttackPower`, `MaxHp`, `Armor`는 `float` 기반으로 구현되어 있다.
- `MaxMana`는 현재 `int` 기반이다.

## 5. 거리 판정
- 사거리는 정수값이다.
- 사거리는 현재 유닛 위치에서 몇 칸 떨어진 적까지 기본 공격이 가능한지를 의미한다.
- 거리 계산은 [HexCoord.cs](D:/Unity/Private/Assets/Scripts/Autobattler/Board/HexCoord.cs)를 기준으로 한다.
- 구현상 거리 계산 함수는 `HexCoord.Distance`를 사용한다.
- 스킬은 기본 공격 사거리와 별개의 규칙으로 동작할 수 있다.
- 기본 공격은 실제 피해 적용 직전에도 다시 사거리 판정을 수행해야 한다.

## 6. 이동
- 유닛은 공격 대상이 사거리 밖에 있을 때 이동을 시도한다.
- 이동은 인접한 타일로 1칸씩만 가능하다.
- 이미 다른 유닛이 점유 중인 타일로는 이동할 수 없다.
- 이동 속도는 음수가 될 수 없다.
- MoveSpeed 기본값은 `1`이다.
- MoveSpeed가 `1`일 때 1칸 이동 시간은 `1초`다.
- 1칸 이동 시간 공식은 아래와 같다.
  - `이동 시간 = 1초 / 이동속도`

현재 구현 반영 사항:
- 이동은 `HexPathfinder` 기반 A* 경로탐색을 사용한다.
- 단순히 현재 거리만 줄어드는 인접칸을 고르는 것이 아니라, 타겟을 공격 가능한 위치까지의 최단 경로를 계산한다.
- 직선 경로가 아군이나 다른 유닛에 의해 막혀 있더라도, 우회 경로가 존재하면 해당 경로의 다음 1칸으로 이동한다.

## 7. 타겟팅 및 이동 우선순위
전투 중 유닛의 타겟팅과 이동은 아래 규칙을 따른다.

1. 현재 타겟이 있다면 해당 타겟을 우선 유지한다.
2. 현재 타겟이 있고 해당 타겟이 사거리 내라면 해당 타겟을 우선 공격한다.
3. 현재 타겟이 사거리 밖이라면, 타겟을 공격 가능한 위치까지의 경로를 탐색하고 그 경로의 다음 1칸으로 이동을 시도한다.
4. 현재 타겟이 사거리 밖이고, 공격 가능한 위치로 가는 경로가 없다면 현재 타겟을 초기화한다.
5. 현재 타겟이 없다면 우선 현재 사거리 내 적이 있는지 확인하고, 있으면 해당 적을 타겟으로 선정한다.
6. 현재 타겟이 없고 사거리 내 적이 없다면 가장 가까운 적을 찾는다.
7. 현재 타겟이 없고 가장 가까운 적이 있다면, 그 적을 공격 가능한 위치까지의 경로를 탐색하고 다음 1칸으로 이동을 시도한다.
8. 이동 이후에는 이전 타겟 유지 여부와 무관하게, 현재 공격 가능한 적이 있다면 그 적을 새로운 타겟으로 선정할 수 있다.

구현 메모:
- 가장 가까운 적 판정은 `HexCoord.Distance`를 사용한다.
- 가장 가까운 적이 여러 명인 경우의 동률 처리 규칙은 추후 필요 시 명시한다.

## 8. 체력
- 체력은 float 값이다.
- 체력이 `0` 이하가 되면 유닛은 즉시 제거된다.
- 체력은 자동 회복되지 않는다.
- 체력은 스킬, 아이템, 기타 특수 효과로만 회복될 수 있다.
- 현재 체력은 최대 체력을 초과할 수 없다.
- 최대 체력을 넘는 회복은 허용되지 않는다.
- 오버힐은 허용되지 않는다.
- 회복량이 최대 체력을 초과하더라도 실제 결과 체력은 MaxHp까지만 회복된다.

현재 구현 반영 사항:
- `HP`는 `float` 네트워크 값으로 구현되어 있다.
- `ReceiveDamage(float)`와 `RestoreHealth(float)`를 통해 피해/회복을 처리한다.
- `RestoreHealth(float)`는 MaxHp를 초과하지 않도록 클램프한다.
- 죽은 유닛은 회복되지 않는다.

## 9. 체력 UI
- 체력은 체력바 형태의 UI로 표시한다.
- 체력바에는 최대 체력 100 단위마다 구분선을 표시한다.
- 구분선 기준은 `100, 200, 300, ...`이다.
- 체력바에 숫자를 직접 표시하지는 않는다.
- UI 상에서 수치 표기가 필요한 경우, 정수 단위로 반올림하여 표시한다.

현재 구현 반영 사항:
- 체력바는 유닛 프리팹 하위의 월드 스페이스 Canvas 기반으로 구현한다.
- 체력바 fill은 `HealthNormalized` 기준으로 갱신한다.
- 체력바 눈금은 MaxHp 기준 100 단위마다 동적으로 생성한다.
- 원본 유닛이 Combat에서 숨김 처리될 때, 체력바도 같이 숨겨진다.
- 체력바는 `Spawned()` 이전에는 Networked 프로퍼티를 읽지 않는다.

## 10. 기본 공격
- 공격력은 float 값이다.
- 공격력이 `0` 이하일 수는 있으나, 음수 피해를 주지는 않는다.
- 기본 공격은 물리 피해다.
- 공격 대상이 사거리 내에 있으면 공격할 수 있다.
- 공격 속도는 float 값이며, 1초 동안 공격 가능한 횟수를 의미한다.
- 기본 공격 간격은 아래와 같다.
  - `공격 간격 = 1초 / AttackSpeed`
- 공격 성공 시 공격자는 ManaGainOnAttack만큼 마나를 획득한다.
- 피격자는 ManaGainOnHit만큼 마나를 획득한다.

현재 구현 반영 사항:
- 기본 공격 적용 직전 `HexCoord.Distance <= AttackRange` 검증을 다시 수행한다.
- 따라서 사거리 밖 대상에게 피해가 들어가지 않아야 한다.

## 11. 방어력 및 피해 계산
- 유닛은 Armor와 MagicResist를 가진다.
- Armor는 물리 피해를 감소 또는 증폭시킨다.
- MagicResist는 특수 피해를 감소 또는 증폭시킨다.
- Armor와 MagicResist는 음수가 될 수 있다.
- 피해 타입은 enum으로 관리한다.

권장 예시:

```csharp
public enum DamageType
{
    Physical,
    Special
}
```

피해 계산식은 아래와 같다.

- 방어 수치가 양수일 경우
  - `FinalDamage = RawDamage / (1 + Defense * 0.01f)`
- 방어 수치가 음수일 경우
  - `FinalDamage = RawDamage * (2 - 1 / (1 - Defense * 0.01f))`

적용 규칙:
- 물리 피해는 Armor를 사용한다.
- 특수 피해는 MagicResist를 사용한다.
- 최종 피해량은 0 이상이어야 한다.

현재 구현 상태:
- 아직 문서의 방어력 공식은 전부 반영되지 않았다.
- 현재는 단순 감소 방식이 일부 코드에 남아 있을 수 있으므로, 이후 별도 구현이 필요하다.

## 12. 마나
- 마나는 float 값이다.
- 기본 공격 시 공격자는 `20`의 마나를 얻는다.
- 피격 시 피격자는 `10`의 마나를 얻는다.
- 마나가 MaxMana 이상이 되면 스킬 사용 조건을 확인한다.
- 스킬 발동 후 남는 마나는 없으며, 현재 마나는 `0`이 된다.
- 초과 마나는 이월하지 않는다.

현재 구현 상태:
- 마나 획득 및 스킬 조건은 아직 문서 기준으로 완전히 반영되지 않았다.

## 13. 스킬
- 각 유닛은 고유 스킬을 가질 수 있다.
- 스킬은 기본 공격과 별개의 시스템이다.
- 스킬은 기본 공격 사거리보다 먼 적을 대상으로 할 수 있다.
- 스킬은 물리 또는 특수 중 하나의 타입을 명확히 가진다.
- 스킬 사용 가능 여부를 먼저 확인해야 한다.
  - 예: 스킬 사거리 내에 적이 있는지
  - 예: 현재 타겟이 유효한지
- 타겟이 없으면 스킬은 시전할 수 없다.
- 스킬 실패 시에도 마나는 소모한다.
- 기본 공격 도중에도 스킬은 사용할 수 있다.
- 스킬 시전 중에는 이동과 기본 공격이 중단된다.

구현 메모:
- 스킬 사용 조건 검증 함수와 실제 스킬 실행 함수는 분리하는 것이 좋다.
- 이후 스킬별 타겟 조건, 범위, 시전 시간, 피해 타입, 부가 효과는 데이터로 분리한다.

## 14. 전투 종료
- 전투는 한쪽 팀의 유닛이 모두 제거되면 종료된다.
- 추후 전투 페이즈 타이머가 추가되면, 타이머 종료 시에도 전투가 종료될 수 있다.
- 현재는 1:1 전투를 기준으로 테스트한다.
- 추후 최대 8명까지 참여하는 구조로 확장한다.
- 다수의 전투가 동시에 진행되는 경우, 가장 늦게 끝난 팀의 전투가 종료된 시점을 기준으로 다음 페이즈가 진행된다.

현재 구현 반영 사항:
- 현재는 각 전투쌍에서 한쪽 팀이 전멸하면 해당 전투쌍이 종료 조건을 만족한다.
- 활성 전투쌍이 모두 종료 조건을 만족하면 라운드를 복구하고 `Result` 상태로 전환한다.
